Compatibilidad
^^^^^^^^^^^^^^

Notas de compatibilidad al momento de esta edición, Agosto de 2017

    + python : cocos requiere python 2.6, 2.7 o 3.3+
      El código base de cocos provee soporte directo para 2.x y 3.x, no es necesario ejecutar 2to3

    + pyglet : pyglet 1.2 o superior (se recomienda 1.3+)

Instalación
^^^^^^^^^^^

Teniendo un setuptools moderno o pip instalado.

Si bien::

    easy_install cocos2d

o::

    pip install cocos2d

debería funcionar, te perderías la documentación y los ejemplos de código. Por lo tanto, es recomendable

    + instalar six y pyglet
    + descargar cocos2d-0.6.5.zip de https://pypi.python.org/pypi/cocos2d/
    + descomprimir en una ubicación conveniente, la carpeta superior será cocos2d-0.6.5
    + la documentación en html estará en cocos2d-0.6.5/doc/html
    + los prácticos ejemplos de código estarán en cocos2d-0.6.5/test, cocos2d-0.6.5/samples, cocos2d-0.6.5/utest
    + instalar con::

        cd cocos2d-0.6.5
        setup.py install

      o en su defecto apuntando a ese directorio en un .pth o PYTHONPATH
    + Todo el contenido de cocos debería estar disponible ahora, con estas posibles excepciones

        - partículas en cocos
        - soporte para sonido SDL
        - soporte para sonido y video AVBin (pyglet)

Si las quieres agregar, consulta la sección de dependencias.


Utilizando PYTHONPATH o .pth
^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Es seguro apuntar a un checkout de cocos desde un .pth o PYTHONPATH tanto
en python 2.x como en 3.x; el mismo checkout se puede especificar al mismo tiempo para los dos python.

Lo mismo vale para pyglet vs 1.3+ (pero ten en cuenta que para pyglet 1.3.0rc1 también deberás
instalar el paquete 'future')

Pyglet en versiones < 1.3 provee compatibilidad con python 3 ejecutando 2to3 durante la instalación, así que mientras
se puede apuntar a un checkout de pyglet desde un .pth o PYTHONPATH en python 2.x,
eso no funcionaría en python 3.
