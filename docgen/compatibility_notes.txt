Cambios en la api y otras notas de compatibilidad


cocos 0.6.5
-----------

    - La primera versión de cocos con soporte para pyglet 1.3, testeada con pyglet 1.3.rc1
	- Esta versión de cocos también se puede ejecutar con pyglet 1.2.x
	- La próxima versión de cocos ya no tendrá soporte para python 2.6
	- El soporte para Python 2.7 se puede terminar en cualquier versión futura de cocos sin aviso previo: si previamente pyglet lo elimina o si una funcionalidad nueva necesita más que una modificación trivial, entonces cocos será compatible solo con py3.


cocos 0.6.4
-----------

Los import en la librería cocos son más específicos, así que el código fuente
que estaba haciendo ``from cocos.somemodule import *`` podría fallar con algunos
símbolos.
Para reparar esos casos, agregar exports explícitos según se requiera.

Las clases MapColliders cambiaron su API para ser más útiles y claras.
    - :class:`~.mapcolliders.RectMapCollider` se movió a su propio módulo,
      :mod:`.mapcolliders`.
    - :class:`~.mapcolliders.RectMapCollider` se renombró como 
      :class:`.RectMapWithPropsCollider`.
    - La nueva :class:`~.mapcolliders.RectMapCollider` controla un caso más
      sencillo de colisión, cuando todos los cuadrados son sólidos: ahora no
      es necesario establecer propiedades de celdas para que un borde no se
      deba pasar por arriba.
    - :class:`.TmxObjectMapCollider` ahora maneja colisiones AABB con cualquier
      :class:`.TmxObject` que se pueda cargar.

:meth:`.Rect.intersects()` ahora solo se interseca si hay superposición de algún
punto interior.

Se eliminaron de :class:`.Scene` los métodos ``push_all_handlers()`` y 
``remove_all_handlers()``.

Se cambiaron los nombres de dos métodos en :class:`.ScrollingManager`, el nombre
antiguo todavía está soportado pero se eliminará en el futuro:

  - ``pixel_from_screen()`` \-> ``screen_to_world()``
  - ``pixel_to_screen()`` \-> ``world_to_screen()``
